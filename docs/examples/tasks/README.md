# Tasks — a runnable taut example

The [Getting Started](../../GettingStarted.md) task-board web API, runnable end to
end with no network and no codegen.

```sh
python3 run.py
```

Output:
```
valid IR: 4 messages, 1 enum(s), 5 methods
exported tasks.ir.json
Task -> 62 bytes: a50101026a7368697020707269736d03010...
round-trips (incl. nested + list-of-messages): True
breaking changes vs export: 0
```

## Files

- `tasks.taut.py` — the authored IR (the only governed artifact).
- `run.py` — load → validate → export → encode/decode a composed value → run the
  breaking-change gate. Uses only the library API.
- `tasks.ir.json` — the neutral IR, generated by `run.py` (what other languages read).
- `generate.py` — emit per-language code into `generated/`.
- `generated/` — **api / client / server** for Python, TypeScript, Rust, and C++
  (`python generate.py` to regenerate). See [generated/README.md](generated/README.md).

## What it demonstrates

- **Message composition** — `Task` embeds an optional `User` and a `List(Comment)`;
  each `Comment` embeds a `User`. The codec nests automatically.
- **Delivery shapes** — `tasks.subscribe` is an **Atom** (whole list, latest-wins);
  `activity.subscribe` is a **Log** (replay then tail).
- **Evolution features** — `Task` declares `reserved=[6, "priority"]` (a retired
  tag + name that can never be reused) and `next_id=7` (the next tag to allocate,
  validated to be above every used/reserved tag).
- **Forward-compat + side-channel** — `run.py` straps a `Trace` extension (from a
  *separate infra schema*) onto a `Task`'s bytes; the app decodes its Task
  obliviously and the side-channel survives in the unknown bag.
